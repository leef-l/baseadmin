# 守护进程管理说明

守护进程管理用于让超级管理员在后台维护宝塔 Supervisor 进程，支持创建、编辑、查看、重启、暂停、删除、批量重启、批量暂停、批量删除和查看日志。

## 功能边界

当前模块只面向超级管理员或被显式授权的高权限角色。

入口：

- 页面：`/system/daemon`
- API 前缀：`/api/system/daemon`
- 表：`system_daemon`
- 后端逻辑：
  - `admin-go/app/system/internal/logic/daemon/daemon.go`
  - `admin-go/app/system/internal/logic/daemon/supervisor.go`
- 前端页面：
  - `vue-vben-admin/apps/web-antd/src/views/system/daemon/index.vue`
  - `vue-vben-admin/apps/web-antd/src/views/system/daemon/modules/form.vue`

## 依赖环境

当前实现按宝塔 Supervisor 插件布局写入配置：

| 内容 | 路径 |
|------|------|
| Supervisor profile | `/www/server/panel/plugin/supervisor/profile/{program}.ini` |
| Supervisor 日志目录 | `/www/server/panel/plugin/supervisor/log/` |
| 宝塔 Supervisor 配置 | `/www/server/panel/plugin/supervisor/config.json` |
| 优先使用的 supervisorctl | `/www/server/panel/pyenv/bin/supervisorctl` |
| 兜底 supervisorctl | `/usr/bin/supervisorctl` |

如果这些路径不存在，创建或操作守护进程会失败。服务器需要安装并启用宝塔 Supervisor 插件。

## 数据表字段

`system_daemon` 保存后台配置和 Supervisor 配置来源。

| 字段 | 含义 |
|------|------|
| `name` | 后台显示名称 |
| `program` | Supervisor 进程名，唯一，建议只用字母、数字、下划线、短横线 |
| `command` | 启动命令 |
| `directory` | 运行目录 |
| `run_user` | 运行用户，默认 `root` |
| `numprocs` | 进程数量 |
| `priority` | Supervisor 启动优先级 |
| `autostart` | 是否随 Supervisor 启动 |
| `autorestart` | 异常退出是否自动重启 |
| `startsecs` | 启动后稳定多少秒算成功 |
| `startretries` | 启动失败重试次数 |
| `stop_signal` | 停止信号，默认 `QUIT` |
| `environment` | Supervisor environment 格式环境变量 |
| `remark` | 备注 |

运行态信息不直接存在表里，而是列表和详情时从 Supervisor 状态里合并：

- `runStatus`
- `pid`
- `uptime`
- `statusText`
- `configPath`
- `outLogPath`
- `errLogPath`

| 详情 | `GET /api/system/daemon/detail` | `system:daemon:view` |

| 更新 | `PUT /api/system/daemon/update` | `system:daemon:update` |
| 删除 | `DELETE /api/system/daemon/delete` | `system:daemon:delete` |
| 批量删除 | `DELETE /api/system/daemon/batch-delete` | `system:daemon:batch-delete` |
| 重启 | `POST /api/system/daemon/restart` | `system:daemon:restart` |
| 批量重启 | `POST /api/system/daemon/batch-restart` | `system:daemon:restart` |

菜单和按钮由迁移 `000013_add_system_daemon` 写入，默认授权给基线超级管理员角色。

## 创建流程

后台创建守护进程时，后端会按顺序执行：

1. 校验字段
2. 校验 `program` 唯一
3. 写入 `system_daemon`
4. 写入 `/www/server/panel/plugin/supervisor/profile/{program}.ini`
5. 更新宝塔 Supervisor `config.json`
6. 执行 `supervisorctl update`
7. 执行 `supervisorctl restart {program}:*`

如果 Supervisor 操作失败，接口会返回错误，避免数据库和实际进程状态静默漂移。

## 编辑流程

编辑时：

1. 先读取原配置
2. 校验新 `program` 是否与其他记录冲突
3. 更新数据库
4. 重写 Supervisor profile
5. 更新宝塔 `config.json`
6. 执行 `supervisorctl reread` / `update`
7. 对目标进程做重启或刷新

如果修改了 `program`，旧 profile 和旧日志会被清理。

## 删除流程

删除时：

1. 尝试停止 Supervisor 进程
2. 删除数据库记录
3. 删除 profile 文件
4. 从宝塔 `config.json` 移除记录
5. 删除 normal/error 日志文件
6. 执行 Supervisor 更新

批量删除会逐个处理，每条失败会返回对应错误结果。

## 日志查看

日志 API 支持读取：

- normal：标准输出日志
- error：标准错误日志

日志文件路径：

```text
/www/server/panel/plugin/supervisor/log/{program}.log
/www/server/panel/plugin/supervisor/log/{program}.err.log
```

后台只读取尾部内容，避免大日志一次性打爆响应。

## 安全注意

- 这是高危功能，不能开放给普通租户或商户管理员。
- `command` 和 `directory` 直接影响服务器进程，只能由可信管理员维护。
- 不要把数据库、第三方密钥等敏感信息写到公开日志里。
- 如果需要环境变量，使用 Supervisor `environment` 格式，并控制后台角色权限。
- 删除守护进程会删除对应 profile 和日志文件，不会删除业务代码目录。

## 排查命令

服务器上可用这些命令排查：

```bash
/www/server/panel/pyenv/bin/supervisorctl status
/www/server/panel/pyenv/bin/supervisorctl reread
/www/server/panel/pyenv/bin/supervisorctl update

ls -la /www/server/panel/plugin/supervisor/profile
ls -la /www/server/panel/plugin/supervisor/log
sed -n '1,220p' /www/server/panel/plugin/supervisor/config.json
```

如果后台列表没有运行态：

1. 检查 `supervisorctl status` 是否可执行
2. 检查 `program` 是否和 profile 文件名一致
3. 检查宝塔 Supervisor 插件是否正常
4. 查看 error 日志确认启动命令是否失败
