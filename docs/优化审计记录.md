# 优化审计记录

## 第 1-50 轮

### 已完成修复

- 后端 `system/auth` 修复“用户不存在但返回空信息”的错误路径，并把角色/权限加载错误从静默吞掉改为显式返回。
- 后端 `system/auth` 的密码变更逻辑补上已删除用户判断。
- 后端 `system/users` 修复详情接口读取角色时忽略数据库错误的问题；列表补齐角色标题填充的错误返回。
- 后端 `system/users`、`upload/file`、`upload/uploader` 将一批 `g.Map + 手写时间字段/软删除` 改为 `DO + Delete()` 写法，降低 GoFrame 约束偏离。
- 上传 COS 存储链路改为沿用请求 `ctx`，不再使用 `context.Background()`，请求取消/超时能正确下传。
- 操作日志异步写入改为 `context.WithoutCancel(ctx)`，保留上下文值，不再直接丢失请求链路信息。
- 前端 `web-antd` 的 `typecheck` 脚本补上 `NODE_OPTIONS=--max-old-space-size=4096`，解决当前环境直接 OOM。
- 前端 `vxe-table` 本地适配层改为导出更贴近页面实际用法的 grid 配置类型，并放宽 `columns` 声明，消除大批旧页面与新类型定义的冲突。
- 前端 CRUD 表单模块统一修复：
  - `modalApi.getData<T | null>()` 改为 `getData<T>()`
  - 表单提交值显式收敛到对应 `Create/Update` 请求类型
- 前端共享组件修复：
  - `menu-transformer` 修正 `RouteMeta.title` 的类型保持
  - 文件管理器树选择事件兼容 `Key[]`
  - 文件管理器弹窗去掉不必要的 `null` 泛型
  - `json-editor` 改用 `Mode.tree`
  - `image-upload` 删除未使用的 `watch`
