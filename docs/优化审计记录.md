## 负载守卫补充（2026-04-10）

- 现有 `scripts/wait-for-cpu-idle.sh` 之前只在任务启动前判断一次 CPU 使用率，任务跑起来以后即使机器被顶到高位，也不会自动让出资源。
- 已新增 `scripts/run-task-with-load-guard.sh` 作为通用负载守卫：启动前先等，运行中如果 CPU 使用率达到 `80%`，会对整组任务发送 `STOP`；等机器回落到 `50%` 再自动 `CONT` 恢复，保证“中途不停任务，只暂停等待”。
- `scripts/run-node-task-with-limits.sh` 与 `scripts/run-go-task-with-limits.sh` 都已切到这层通用守卫，继续保留 `systemd-run` 的 `MemoryMax` / `CPUQuota` / `TasksMax` 限额，同时在非 systemd 分支保留 `nice` / `ionice` 的降优先级执行。
- 默认恢复阈值已统一到 `50%`，与服务器铁律保持一致，避免长任务在高位刚恢复就再次把机器顶满。
