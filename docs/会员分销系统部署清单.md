# fund-disk 会员分销系统部署清单

部署 / 升级前对照本清单逐项检查。

## 1. 数据库

数据库：`sql_funddisk_easytestdev_online @ 127.0.0.1:3306`

按版本顺序应用迁移（golang-migrate 或手动）：

```bash
mysql -u <user> -p<pass> sql_funddisk_easytestdev_online < admin-go/database/migrations/000017_add_member_app.up.sql
mysql -u <user> -p<pass> sql_funddisk_easytestdev_online < admin-go/database/migrations/000018_add_member_business_config_and_purchase_limit.up.sql
mysql -u <user> -p<pass> sql_funddisk_easytestdev_online < admin-go/database/migrations/000019_add_member_contract.up.sql
```

> 000018/000019 都已做幂等化处理，可重复执行不会破坏数据。

应用后核对：
- `SELECT id,name,daily_purchase_limit FROM member_level;`  → 看到 普通=1 / 高级=2 / 核心=5
- `SELECT id,config_key FROM member_business_config;`  → 至少 1 行 `global`
- `SHOW TABLES LIKE 'member_contract%';`  → `member_contract` + `member_contract_template`

## 2. 配置文件

复制 example 到正式 config，按环境填值：

```bash
cd admin-go/app/member/manifest/config
cp config.yaml.example config.yaml
vim config.yaml
```

**必填项**：
- `database.default.link` — 数据库连接串
- `redis.default.address` / `redis.default.pass` — Redis 配置
- `jwt.secret` / `jwt.memberSecret` — 必须 ≥32 字符，两者**必须不同**
- `sms.providers.aliyun.accessKeyId` / `accessKeySecret` / `signName` / `templateCode` — 生产必须改用阿里云通道
- `sms.provider: aliyun` — 切到 aliyun（默认是 mock）

**可调项**（默认值即可工作）：
- `member.h5BaseURL` — 邀请链接基址
- `member.contractStorageDir` — 合同 PDF/HTML 落盘目录（默认 `./runtime/contracts`，已自动转绝对路径）
- `member.contractChromiumPath` — chromium 二进制路径（留空自动 PATH 探测）
- `member.teamExportRoot` — 团队数据导出目录（默认 `./resource/team-export`，自动转绝对路径）

## 3. Chromium（合同 PDF 生成）

会员合同要走真 PDF 生成，依赖 headless Chromium。

```bash
# Rocky/CentOS 9
dnf install -y ungoogled-chromium

# Ubuntu/Debian
apt install -y chromium-browser
```

验证可用：
```bash
ungoogled-chromium --version    # 或 chromium / chromium-browser / google-chrome
```

无需 GUI。如果服务器是非 root 用户运行 member 服务，需要：
- 确保 Chromium 用户能写入 `member.contractStorageDir`
- 服务进程能 fork 子进程
- 可保留 `--no-sandbox`（chromium headless 需要），或在容器中通过 user namespace 隔离

## 4. SMS 通道

切到生产时把 `sms.provider` 从 `mock` 改为 `aliyun`，并在 `sms.providers.aliyun.*` 下填写：
- 阿里云 RAM 子账号 AK/SK
- 已审批的签名 `signName`
- 已审批的模板 `templateCode`
- 模板必须包含 `${code}` 占位符

不切则 mock 通道固定返回 `123456`，仅供本地联调。

## 5. 启动

```bash
cd admin-go
# 后台进程或 systemd
GO111MODULE=on go run app/member/main.go
# 或编译后跑
go build -o /www/wwwroot/funddisk.easytestdev.online/member/member ./app/member
nohup /www/wwwroot/funddisk.easytestdev.online/member/member > member.log 2>&1 &
```

启动后核对端口：
```bash
ss -tlnp | grep 10027
curl -s http://127.0.0.1:10027/api/member-portal/biz-config | head
```

## 6. 前端 H5

```bash
cd h5-react
cp .env.example .env
vim .env       # 调整 VITE_API_PROXY_TARGET 与 VITE_API_BASE
../scripts/run-node-task-with-limits.sh npm install
../scripts/run-node-task-with-limits.sh npm run build
# dist/ 由 Nginx 托管，/api 反代到后端 10027
```

Nginx 反代示例见 `h5-react/README.md`。

## 7. 定时任务

cron 在 member 服务进程内通过 gcron 注册：
- 每天 01:30 扫描会员等级过期
- 每小时一次轻量补扫
- 每天 00:05 重置 `today_purchase_count`

启动 member 服务即生效，无需 systemd timer。

## 8. 升级注意

升级 member 服务前：
1. `git pull` 后查看 `database/migrations/` 是否有新文件
2. 应用迁移（已幂等）
3. 重启 member 服务
4. 验证 portal API：`/api/member-portal/biz-config` 返回 200

## 9. 监控点

- 钱包流水 `member_wallet_log` 暴增（自购返佣 + 直推返佣）
- 合同 PDF 生成失败率：`SELECT pdf_status, COUNT(*) FROM member_contract GROUP BY pdf_status;`
- 团队导出失败：`SELECT COUNT(*) FROM member_team_export WHERE deploy_status=3;`
- 进货时间窗外的 4xx 请求：日志中"当前不在进货时间内"
