# 生产运维说明

当前生产环境目标：

- 域名：`baseadmin.easytestdev.online`
- 部署目录：`/www/wwwroot/baseadmin.easytestdev.online`
- 管理方式：`systemd + Nginx`，后台也支持通过宝塔 Supervisor 模块管理守护进程
- 后端服务：`baseadmin-system`、`baseadmin-upload`

## 服务与端口

- `system`：`127.0.0.1:10022`
- `upload`：`127.0.0.1:10023`
- `frontend`：静态文件，由 Nginx 对外提供
- `MySQL`：`127.0.0.1:3306`
- `Redis`：`127.0.0.1:6379`
- `adminer`：`127.0.0.1:10025`

## 数据库铁律

- 当前站点唯一联调/迁移/冒烟库是 `baseadmin.easytestdev.online` 对应的 MySQL 库：`127.0.0.1:3306/sql_baseadmin_easytestdev_online`。
- 后端连接真源是 `/www/wwwroot/baseadmin.easytestdev.online/system/manifest/config/config.yaml`，仓库本机直跑时对应 `admin-go/app/{system,upload,demo}/manifest/config/config.yaml` 和 `admin-go/.env`。
- 在当前服务器测试 demo、执行 `migrate up`、写入种子数据或做接口冒烟时，不允许改用临时库、Docker 开发库、其它项目库或其它域名数据库。
- 任何数据库操作前先确认库名；涉及结构或初始化数据变更必须先补 `admin-go/database/migrations/`。

## 服务资源限制

GitHub Actions 在后端部署时会重写 `baseadmin-system`、`baseadmin-upload` 的 unit 文件，并带上默认资源限额：

- `baseadmin-system`：`MemoryHigh=384M`、`MemoryMax=640M`、`CPUQuota=80%`
- `baseadmin-upload`：`MemoryHigh=256M`、`MemoryMax=384M`、`CPUQuota=50%`
- 两个服务共同配置：`TasksMax=512`、`Nice=5`，并通过 `OOMScoreAdjust` 降低把整机拖死的风险

要确认线上实际生效的 unit 内容，直接看：

```bash
systemctl cat baseadmin-system
systemctl cat baseadmin-upload

systemctl show baseadmin-system -p MemoryHigh -p MemoryMax -p TasksMax -p Nice -p OOMScoreAdjust -p CPUQuotaPerSecUSec
systemctl show baseadmin-upload -p MemoryHigh -p MemoryMax -p TasksMax -p Nice -p OOMScoreAdjust -p CPUQuotaPerSecUSec
```

注意：修改 `.github/workflows/deploy.yml` 后，需要再执行一次后端部署，远端 unit 文件才会更新。

## 自定义域名与 SSL

后台 `系统管理 -> 域名管理` 可以为租户或商户绑定自定义域名，并一键写入宝塔 Nginx 配置、申请 SSL 证书。

相关路径：

| 内容 | 路径 |
|------|------|
| 托管 Nginx 配置 | `/www/server/panel/vhost/nginx/baseadmin-managed-{domain}.conf` |
| 证书目录 | `/www/server/panel/vhost/cert/{domain}/` |
| 证书文件 | `fullchain.pem`、`privkey.pem` |
| ACME challenge 根目录 | `/www/wwwroot/baseadmin.easytestdev.online/admin/.well-known/acme-challenge/` |

排查命令：

```bash
nginx -t
nginx -s reload
ls -la /www/server/panel/vhost/nginx | rg 'baseadmin-managed'
ls -la /www/server/panel/vhost/cert
curl -I https://your-domain.example
```

后台申请 SSL 前必须确保域名已经解析到当前服务器。泛解析可用，但 Let's Encrypt 仍需要 HTTP challenge 能通过公网访问。

访问自定义域名时，`system` 服务会按 `Host` / `X-Forwarded-Host` 匹配 `system_domain`，并校验当前 JWT 的租户/商户归属。不属于该域名归属的账号会被拒绝。

## 宝塔 Supervisor 守护进程

后台 `系统管理 -> 守护进程` 支持创建、编辑、查看、重启、暂停、删除和批量操作宝塔 Supervisor 进程。该功能应只授权给超级管理员或可信运维角色。

相关路径：

| 内容 | 路径 |
|------|------|
| Supervisor profile | `/www/server/panel/plugin/supervisor/profile/{program}.ini` |
| Supervisor 日志 | `/www/server/panel/plugin/supervisor/log/` |
| 宝塔 Supervisor 配置 | `/www/server/panel/plugin/supervisor/config.json` |
| supervisorctl | `/www/server/panel/pyenv/bin/supervisorctl` |

排查命令：

```bash
/www/server/panel/pyenv/bin/supervisorctl status
/www/server/panel/pyenv/bin/supervisorctl reread
/www/server/panel/pyenv/bin/supervisorctl update

ls -la /www/server/panel/plugin/supervisor/profile
ls -la /www/server/panel/plugin/supervisor/log
sed -n '1,220p' /www/server/panel/plugin/supervisor/config.json
```

详细字段和操作流程见 [守护进程管理说明](守护进程管理说明.md)。

## GitHub Actions 部署

当前仓库通过 `.github/workflows/deploy.yml` 在 `push main` 时自动部署。

仓库级 GitHub Actions secret 至少需要：

- `DEPLOY_SSH_KEY`：用于 SSH 登录 `root@baseadmin.easytestdev.online` 的私钥
- `SMOKE_USERNAME`：部署后线上冒烟账号用户名
- `SMOKE_PASSWORD`：部署后线上冒烟账号密码

常用查看命令：

```bash
gh run list --workflow deploy.yml --limit 5
gh run view <run-id>
gh run watch <run-id> --exit-status
```

工作流中的后端部署动作会按以下顺序执行：

1. `systemctl stop baseadmin-system` / `systemctl stop baseadmin-upload`
2. 替换远端二进制
3. 执行数据库迁移
4. 自动确保专用 smoke 角色/账号存在
5. 自动回填历史 `upload_file.dir_id`
6. `systemctl start ...`
7. 检查 `http://127.0.0.1:10022/readyz` 和 `http://127.0.0.1:10023/readyz`
8. 用 smoke 账号做真实登录、目录规则、上传、公网 URL 冒烟

如果需要在服务器上手工执行相同操作：

```bash
/www/wwwroot/baseadmin.easytestdev.online/bin/ensure-smoke-user \
  --config /www/wwwroot/baseadmin.easytestdev.online/system/manifest/config/config.yaml \
  --username "$SMOKE_USERNAME" \
  --password "$SMOKE_PASSWORD"

/www/wwwroot/baseadmin.easytestdev.online/bin/upload-backfill \
  --config /www/wwwroot/baseadmin.easytestdev.online/system/manifest/config/config.yaml \
  --dry-run
```
