# 生产运维说明

当前生产环境目标：

- 域名：`baseadmin.easytestdev.online`
- 部署目录：`/www/wwwroot/baseadmin.easytestdev.online`
- 管理方式：`systemd + Nginx`
- 后端服务：`baseadmin-system`、`baseadmin-upload`

## 服务与端口

- `system`：`127.0.0.1:10022`
- `upload`：`127.0.0.1:10023`
- `frontend`：静态文件，由 Nginx 对外提供
- `MySQL`：`127.0.0.1:10020`
- `Redis`：`127.0.0.1:10021`
- `adminer`：`127.0.0.1:10025`

## 常用 systemd 命令

```bash
systemctl status baseadmin-system --no-pager -l
systemctl status baseadmin-upload --no-pager -l

systemctl start baseadmin-system
systemctl start baseadmin-upload

systemctl stop baseadmin-system
systemctl stop baseadmin-upload

systemctl restart baseadmin-system
systemctl restart baseadmin-upload

systemctl enable baseadmin-system
systemctl enable baseadmin-upload

systemctl disable baseadmin-system
systemctl disable baseadmin-upload
```

## 日志命令

```bash
journalctl -u baseadmin-system -f
journalctl -u baseadmin-upload -f

journalctl -u baseadmin-system -n 200 --no-pager
journalctl -u baseadmin-upload -n 200 --no-pager
```

## 健康检查

```bash
curl -fsS http://127.0.0.1:10022/healthz
curl -fsS http://127.0.0.1:10022/readyz
curl -fsS http://127.0.0.1:10023/healthz
curl -fsS http://127.0.0.1:10023/readyz
```

## GitHub Actions 部署

当前仓库通过 `.github/workflows/deploy.yml` 在 `push main` 时自动部署。

常用查看命令：

```bash
gh run list --workflow deploy.yml --limit 5
gh run view <run-id>
gh run watch <run-id> --exit-status
```

工作流中的后端部署动作会按以下顺序执行：

1. `systemctl stop baseadmin-system` / `systemctl stop baseadmin-upload`
2. 替换远端二进制
3. `systemctl start ...`
4. 检查 `http://127.0.0.1:10022/readyz` 和 `http://127.0.0.1:10023/readyz`

## 最常用的一组命令

```bash
systemctl restart baseadmin-system
systemctl restart baseadmin-upload

journalctl -u baseadmin-system -f
journalctl -u baseadmin-upload -f

curl -fsS http://127.0.0.1:10022/readyz
curl -fsS http://127.0.0.1:10023/readyz
```
