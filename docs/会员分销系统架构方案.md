# 会员分销系统架构方案

## 一、项目总览

基于 GBaseAdmin 框架，新增 `member` 应用，实现多级分销 + 寄售交易 + 站点裂变功能。

### 核心业务

1. 无限层级会员树（上下级关系链 + 换绑上级）
2. 会员等级晋升（有效用户数 + 团队营业额 + 时限天数）
3. 三钱包体系（优惠券余额、奖金余额、推广奖余额）
4. 商城商品（优惠券余额购买）
5. 仓库商品（寄售模式，自动加价流转，线下确认交易）
6. 团队数据提取 + 独立站点部署（最高等级触发裂变）

---

## 二、应用规划

| 应用 | 用途 | 后端端口 |
|------|------|----------|
| `system` | 系统权限（已有） | 10022 |
| `upload` | 文件上传（已有） | 10023 |
| `member` | **会员分销系统（新增）** | 10027 |

`member` 应用涵盖：会员管理、等级管理、钱包管理、商城商品、仓库商品、寄售交易、团队数据提取。

### 3.1 会员用户表 `member_user`

会员体系的核心表，记录会员的层级关系、等级、激活状态。

```sql
CREATE TABLE `member_user` (
  `id` bigint unsigned NOT NULL COMMENT '会员ID（Snowflake）',
  `parent_id` bigint unsigned NOT NULL DEFAULT '0' COMMENT '上级会员ID，0表示顶级',
  `username` varchar(50) NOT NULL COMMENT '用户名（登录账号）|search:eq|keyword:on|priority:100',
  `password` varchar(255) NOT NULL COMMENT '密码（bcrypt加密）',
  `nickname` varchar(50) DEFAULT NULL COMMENT '昵称|search:like|keyword:on|priority:95',
  `phone` varchar(20) DEFAULT NULL COMMENT '手机号|search:eq|keyword:on|priority:90',
  `avatar` varchar(500) DEFAULT NULL COMMENT '头像',
  `real_name` varchar(50) DEFAULT NULL COMMENT '真实姓名|search:like|keyword:on',
  `level_id` bigint unsigned NOT NULL DEFAULT '0' COMMENT '当前等级|ref:member_level.name|search:select',
  `level_expire_at` datetime DEFAULT NULL COMMENT '等级到期时间',
  `team_count` int unsigned NOT NULL DEFAULT '0' COMMENT '团队总人数',
  `direct_count` int unsigned NOT NULL DEFAULT '0' COMMENT '直推人数',
  `active_count` int unsigned NOT NULL DEFAULT '0' COMMENT '有效用户数（直推+间推已激活）',
  `team_turnover` bigint unsigned NOT NULL DEFAULT '0' COMMENT '团队总营业额（分）',
  `is_active` tinyint(1) unsigned NOT NULL DEFAULT '1' COMMENT '是否激活:0=未激活,1=已激活|search:select',
  `is_qualified` tinyint(1) unsigned NOT NULL DEFAULT '1' COMMENT '仓库资格:0=已失效,1=有效|search:select',
  `invite_code` varchar(32) DEFAULT NULL COMMENT '邀请码|search:eq',
  `register_ip` varchar(45) DEFAULT NULL COMMENT '注册IP',
  `last_login_at` datetime DEFAULT NULL COMMENT '最后登录时间',
  `remark` varchar(500) DEFAULT NULL COMMENT '备注|search:off',
  `sort` int NOT NULL DEFAULT '0' COMMENT '排序（升序）',
  `status` tinyint(1) NOT NULL DEFAULT '1' COMMENT '状态:0=冻结,1=正常|search:select',
  `tenant_id` bigint unsigned NOT NULL DEFAULT '0' COMMENT '租户|ref:system_tenant.name|search:select',
  `merchant_id` bigint unsigned DEFAULT '0' COMMENT '商户|ref:system_merchant.name|search:select',
  `created_by` bigint unsigned DEFAULT NULL COMMENT '创建人ID',
  `dept_id` bigint unsigned DEFAULT NULL COMMENT '所属部门ID',
  `created_at` datetime DEFAULT NULL COMMENT '创建时间',
  `updated_at` datetime DEFAULT NULL COMMENT '更新时间',
  `deleted_at` datetime DEFAULT NULL COMMENT '软删除时间',
  PRIMARY KEY (`id`),
  UNIQUE KEY `uk_username` (`username`),
  UNIQUE KEY `uk_invite_code` (`invite_code`),
  KEY `idx_parent_id` (`parent_id`),
  KEY `idx_level_id` (`level_id`),
  KEY `idx_phone` (`phone`),
  KEY `idx_tenant_id` (`tenant_id`),
  KEY `idx_deleted_at` (`deleted_at`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci COMMENT='会员用户表';
```

**设计要点：**
- `parent_id` 实现无限层级树结构，codegen 自动生成树形接口
- `level_expire_at` 记录等级到期时间，定时任务检查过期后标记 `is_qualified=0`
- `active_count` / `team_turnover` 用于自动升级判断
- `invite_code` 唯一邀请码，用于推广注册

```sql
  KEY `idx_deleted_at` (`deleted_at`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci COMMENT='会员等级配置表';
```

**设计要点：**
- `level_no` 数字越大等级越高，用于自动升级比较
- `duration_days` 配置每个等级的有效期天数
- `need_active_count` + `need_team_turnover` 两个升级条件同时满足才升级
- `is_top=1` 标记最高等级，到达后触发团队数据提取和站点部署

### 3.4 会员钱包表 `member_wallet`

每个会员 3 个钱包，使用 `wallet_type` 区分。

```sql
CREATE TABLE `member_wallet` (
  `id` bigint unsigned NOT NULL COMMENT 'ID（Snowflake）',
  `user_id` bigint unsigned NOT NULL COMMENT '会员|ref:member_user.nickname|search:select',
  `wallet_type` tinyint unsigned NOT NULL DEFAULT '1' COMMENT '钱包类型:1=优惠券余额,2=奖金余额,3=推广奖余额|search:select',
  `balance` bigint NOT NULL DEFAULT '0' COMMENT '当前余额（分）',
  `total_income` bigint unsigned NOT NULL DEFAULT '0' COMMENT '累计收入（分）',
  `total_expense` bigint unsigned NOT NULL DEFAULT '0' COMMENT '累计支出（分）',
  `frozen_amount` bigint unsigned NOT NULL DEFAULT '0' COMMENT '冻结金额（分）',
  `status` tinyint(1) NOT NULL DEFAULT '1' COMMENT '状态:0=冻结,1=正常|search:select',
  `tenant_id` bigint unsigned NOT NULL DEFAULT '0' COMMENT '租户|ref:system_tenant.name|search:select',
  `merchant_id` bigint unsigned DEFAULT '0' COMMENT '商户|ref:system_merchant.name|search:select',
  `created_by` bigint unsigned DEFAULT NULL COMMENT '创建人ID',
  `dept_id` bigint unsigned DEFAULT NULL COMMENT '所属部门ID',
  `created_at` datetime DEFAULT NULL COMMENT '创建时间',
  `updated_at` datetime DEFAULT NULL COMMENT '更新时间',
  `deleted_at` datetime DEFAULT NULL COMMENT '软删除时间',
  PRIMARY KEY (`id`),
  UNIQUE KEY `uk_user_type` (`user_id`, `wallet_type`),
  KEY `idx_tenant_id` (`tenant_id`),
  KEY `idx_deleted_at` (`deleted_at`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci COMMENT='会员钱包表';
```

**设计要点：**
- 每个会员创建时自动生成 3 条钱包记录
- `wallet_type=1` 优惠券余额：可购买商城商品
- `wallet_type=2` 奖金余额：不可购买商品
- `wallet_type=3` 推广奖余额：不可购买商品
- `balance` 使用分为单位，防止浮点精度问题

```sql
  KEY `idx_deleted_at` (`deleted_at`)
```


**完整交易流程：**

```
1. 后台创建仓库商品（init_price=100元） → 分配给会员A
2. 会员A挂卖 → 自动加价10%后 listing_price=110元
   → 生成 listing 记录（挂卖中）
3. 会员B下单购买（不需要付款，线下给钱）
   → 生成 trade 记录（待卖家确认）
   → 商品 goods_status 变为"交易中"
4. 会员A确认交易（表示已线下收到钱）
   → trade_status 变为"已确认完成"
   → 商品 owner_id 变更为会员B
   → 商品 current_price 更新为 110元
   → 平台扣除 5%（5.5元），卖家实收104.5元
   → listing_status 变为"已售出"
   → 商品 goods_status 变回"持有中"
   → trade_count +1
5. 会员B可继续挂卖 → 自动加价10% → listing_price=121元
   → 循环...
```

## 六、非 CRUD 业务逻辑（手写）

以下逻辑无法由 codegen 自动生成，需要在 `member` 应用中手写：

### 6.1 会员注册/登录
- 通过邀请码注册，自动绑定上级
- 注册时自动创建 3 个钱包
- 生成唯一邀请码

### 6.2 等级自动升级（定时任务/事件驱动）
- 检查每个会员的 `active_count` 和 `team_turnover`
- 满足下一等级条件 → 自动升级 + 重置过期时间
- 到期未升级 → `is_qualified=0`，禁止购买仓库商品
- 升到最高等级 → 触发团队数据导出

### 6.3 会员换绑上级
- 后台操作，记录日志
- 需要重新计算原上级和新上级的团队人数/有效用户数

### 6.4 仓库商品分配
- 后台创建商品 → 选择分配给指定会员
- 分配后 `owner_id` 写入该会员

### 6.5 仓库商品交易流程
- 买家下单（不付款）→ 生成交易记录（待确认）
- 卖家确认 → 商品转移 + 自动加价 + 平台扣费
- 检查买家 `is_qualified`，不合格拒绝下单

### 6.6 团队数据提取
- 递归查询目标会员的所有下级
- 导出相关的 member_user、wallet、trade 等数据为 SQL
- 打包成压缩包供下载
- 配合 SaaS 站点部署

会员前端（移动端）独立于 vue-vben-admin 后台，建议：

| 方案 | 技术栈 | 说明 |
|------|--------|------|
